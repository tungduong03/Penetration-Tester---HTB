# Virtual Hosts

Một web server có thể host nhiều website khác nhau trên cùng một IP thông qua cơ chế Virtual Host (VHost).

Ví dụ:

203.0.113.10\
    ├── www.company.com\
    ├── dev.company.com\
    ├── admin.company.com\
    └── test.company.com\

Tất cả đều trỏ tới cùng IP: 203.0.113.10

Nhưng server vẫn biết website nào cần trả về nhờ HTTP Header: 

`Host: www.company.com`

Ví dụ trong apache 

```apache
<VirtualHost *:80>
    ServerName www.company.com
    DocumentRoot /var/www/main
</VirtualHost>

<VirtualHost *:80>
    ServerName admin.company.com
    DocumentRoot /var/www/admin
</VirtualHost>
```

## Sự khác nhau giữa Subdomain và VHost

### 1. Subdomain là khái niệm DNS

Ví dụ:

```
blog.company.com
admin.company.com
mail.company.com
```

Đây là các subdomain của:

`company.com` 

Muốn tồn tại thì thường phải có DNS Record:

```bash
blog.company.com    A    1.2.3.4
admin.company.com   A    1.2.3.4
```

Tra cứu được bằng:

`dig blog.company.com`

hoặc

`host blog.company.com`

### 2. VHost là khái niệm Web Server

VHost chỉ là cấu hình bên trong Apache/Nginx.

Ví dụ:

```apache 
ServerName dev.company.com
``` 

hoặc

```apache 
ServerName staging.company.com
```

Nhưng DNS có thể không hề tồn tại:

❌ Không có DNS Record

## Virtual Host Discovery Tools

- gobuster

```bash 
gobuster vhost -u http://<target_IP_address> -w <wordlist_file> --append-domain
```

Ví dụ 

```bash
gobuster vhost -u http://inlanefreight.htb:81 -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt --append-domain
```
