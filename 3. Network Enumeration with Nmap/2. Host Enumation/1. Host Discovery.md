# Host discovery

## Scan Network Range

```bash
reDrose18@htb[/htb]$ sudo nmap 10.129.2.0/24 -sn -oA tnet | grep for | cut -d" " -f5

10.129.2.4
10.129.2.10
10.129.2.11
10.129.2.18
10.129.2.19
10.129.2.20
10.129.2.28
```

- `10.129.2.0/24`	Target network range.

- `-sn` 	Disables port scanning. (Tắt việc scan port)

- `-oA tnet`	Stores the results in all formats starting with the name 'tnet'.

Phương pháp quét này chỉ hoạt động nếu tường lửa của các máy chủ cho phép. Nếu không, chúng ta có thể sử dụng các kỹ thuật quét khác để tìm hiểu xem các máy chủ có đang hoạt động hay không. Chúng ta sẽ xem xét kỹ hơn các kỹ thuật này trong phần "`Firewall and IDS Evasion`".

## Scan IP List

Tạo list IP

```bash
reDrose18@htb[/htb]$ cat hosts.lst

10.129.2.4
10.129.2.10
10.129.2.11
10.129.2.18
10.129.2.19
10.129.2.20
10.129.2.28
```

Scan bằng nmap 

```bash
reDrose18@htb[/htb]$ sudo nmap -sn -oA tnet -iL hosts.lst | grep for | cut -d" " -f5

10.129.2.18
10.129.2.19
10.129.2.20
```

`-iL` Thực hiện các thao tác quét được xác định đối với các mục tiêu trong danh sách 'hosts.lst' được cung cấp.

## Scan Multiple IPs

Ta cũng có thể ko dùng list mà liệt kê trực tiếp trên câu lệnh nmap 

```bash
reDrose18@htb[/htb]$ sudo nmap -sn -oA tnet 10.129.2.18 10.129.2.19 10.129.2.20| grep for | cut -d" " -f5

10.129.2.18
10.129.2.19
10.129.2.20
```

Nếu các địa chỉ nằm cạnh nhau ta có thể dùng range

```bash
reDrose18@htb[/htb]$ sudo nmap -sn -oA tnet 10.129.2.18-20| grep for | cut -d" " -f5

10.129.2.18
10.129.2.19
10.129.2.20
```
