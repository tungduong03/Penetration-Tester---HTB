# MySQL

## 1. Tổng quan

**MySQL** là hệ quản trị cơ sở dữ liệu quan hệ (**RDBMS**) mã nguồn mở do Oracle phát triển.

Đặc điểm:

* Lưu trữ dữ liệu theo **Database → Table → Row → Column**.
* Sử dụng ngôn ngữ **SQL** để truy vấn và quản lý dữ liệu.
* Hoạt động theo mô hình **Client - Server**.
* Thường dùng cho website và ứng dụng web.

Các file sao lưu CSDL thường có định dạng:

```text
database.sql
wordpress.sql
backup.sql
```

---

## 2. MySQL Client và Server

### MySQL Server

Là thành phần quản lý cơ sở dữ liệu:

* Lưu trữ dữ liệu.
* Xử lý truy vấn.
* Quản lý người dùng và phân quyền.

### MySQL Client

Là chương trình kết nối tới MySQL Server để:

* Truy vấn dữ liệu (`SELECT`)
* Thêm dữ liệu (`INSERT`)
* Cập nhật (`UPDATE`)
* Xóa (`DELETE`)
* Quản lý Database

Nhiều Client có thể truy cập cùng lúc.

---

## 3. Ứng dụng của MySQL

MySQL thường được sử dụng cho:

* Website động.
* CMS (WordPress, Joomla...).
* Web Application.
* Hệ thống quản lý người dùng.

Thường kết hợp với:

| Stack    | Thành phần                   |
| -------- | ---------------------------- |
| **LAMP** | Linux + Apache + MySQL + PHP |
| **LEMP** | Linux + Nginx + MySQL + PHP  |

Dữ liệu thường lưu trong MySQL:

* Username
* Password (thường đã được hash)
* Email
* Customer
* User Information
* Permissions
* Admin
* Nội dung website
* Meta data
* Links
* Form data

---

## 4. SQL Commands

MySQL sử dụng SQL để:

* Truy vấn dữ liệu.
* Thêm, sửa, xóa dữ liệu.
* Quản lý User.
* Tạo/Xóa Database.
* Thay đổi cấu trúc bảng.
* Tạo Index và Relationship.

Trong quá trình xử lý, nếu truy vấn lỗi, Web Application có thể trả về **SQL Error**.

Các lỗi này rất quan trọng trong Pentest vì có thể:

* Tiết lộ DBMS đang sử dụng.
* Rò rỉ tên bảng, cột.
* Xác nhận SQL Injection.
* Tiết lộ cấu trúc Database.

---

## 5. MariaDB

**MariaDB** là một **fork** của MySQL.

* Tương thích gần như hoàn toàn với MySQL.
* Được tạo sau khi Oracle mua lại MySQL.
* Hiện được sử dụng rất phổ biến trên Linux.

---

# 6. Default Configuration

File cấu hình phổ biến:

```bash
/etc/mysql/mysql.conf.d/mysqld.cnf
```

Một số cấu hình quan trọng:

| Cấu hình     | Ý nghĩa                        |
| ------------ | ------------------------------ |
| `port=3306`  | Cổng mặc định của MySQL.       |
| `socket`     | Unix Socket để kết nối nội bộ. |
| `user=mysql` | User chạy dịch vụ MySQL.       |
| `datadir`    | Thư mục lưu Database.          |
| `tmpdir`     | Thư mục file tạm.              |
| `basedir`    | Thư mục cài đặt MySQL.         |

---

# 7. Dangerous Settings

Một số cấu hình ảnh hưởng trực tiếp đến bảo mật:

| Cấu hình           | Nguy cơ                                                                                                                      |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| `user`             | User chạy MySQL, nếu cấu hình sai có thể gây leo thang đặc quyền.                                                            |
| `password`         | Mật khẩu MySQL lưu dạng Plaintext trong file cấu hình nếu phân quyền không đúng.                                             |
| `admin_address`    | Địa chỉ IP cho giao diện quản trị; cấu hình sai có thể cho phép truy cập trái phép.                                          |
| `debug`            | Hiển thị thông tin debug, dễ làm lộ thông tin hệ thống.                                                                      |
| `sql_warnings`     | Hiển thị cảnh báo SQL chi tiết, hỗ trợ attacker thu thập thông tin.                                                          |
| `secure_file_priv` | Giới hạn thao tác `LOAD DATA` và `SELECT ... INTO OUTFILE`; cấu hình không an toàn có thể dẫn đến đọc/ghi file trên máy chủ. |

---

# Ý chính cần nhớ (Pentest)

* **MySQL** là DBMS phổ biến, mặc định sử dụng **port 3306**.
* Hoạt động theo mô hình **Client - Server** và quản lý dữ liệu bằng **SQL**.
* **MariaDB** là phiên bản tương thích của MySQL, thường gặp trên Linux.
* Nếu đọc được file cấu hình (`mysqld.cnf`) hoặc có shell trên máy chủ, có thể thu được:

  * Username MySQL.
  * Password.
  * Đường dẫn Database.
  * Thông tin cấu hình quan trọng.
* Các lỗi SQL hoặc chế độ **debug** có thể tiết lộ nhiều thông tin, hỗ trợ khai thác **SQL Injection**.
* Cần đặc biệt chú ý cấu hình **`secure_file_priv`**, vì nó ảnh hưởng đến khả năng đọc/ghi file thông qua MySQL.
