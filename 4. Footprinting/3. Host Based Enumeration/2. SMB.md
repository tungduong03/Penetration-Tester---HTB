# SMB

## SAMBA

Samba là phần mềm mã nguồn mở cho phép:

- Linux
- Unix

giao tiếp với Windows thông qua SMB.

Ví dụ:

```
Linux
   │
 Samba
   │
SMB Protocol
   │
Windows
```

Nhờ Samba:

- Linux có thể tạo SMB Server.
- Linux truy cập Shared Folder của Windows.
- Windows truy cập Shared Folder của Linux.

**CIFS (Common Internet File System)** là phiên bản cũ của SMB, gần tương ứng với SMBv1.

Đặc điểm:

- Do Microsoft phát triển.
- Hiện nay đã lỗi thời.
- Có nhiều lỗ hổng (ví dụ EternalBlue).

Trong thực tế:

CIFS ≈ SMBv1

### Các cổng SMB

SMB cũ (NetBIOS)

| Port        | Chức năng            |
| ----------- | -------------------- |
| TCP/UDP 137 | NetBIOS Name Service |
| UDP 138     | NetBIOS Datagram     |
| TCP 139     | NetBIOS Session      |


SMB mới

| Port    | Chức năng                  |
| ------- | -------------------------- |
| TCP 445 | SMB trực tiếp (Direct SMB) |


### Các phiên bản SMB

| Phiên bản        | Hệ điều hành                 | Đặc điểm                                                           |
| ---------------- | ---------------------------- | ------------------------------------------------------------------ |
| **CIFS (SMBv1)** | Windows NT 4.0               | Giao tiếp qua NetBIOS, nhiều lỗ hổng, không nên dùng.              |
| **SMB 1.0**      | Windows 2000                 | Hỗ trợ kết nối trực tiếp qua TCP.                                  |
| **SMB 2.0**      | Windows Vista / Server 2008  | Hiệu năng tốt hơn, cải thiện Message Signing và Cache.             |
| **SMB 2.1**      | Windows 7 / Server 2008 R2   | Thêm cơ chế khóa file (Locking).                                   |
| **SMB 3.0**      | Windows 8 / Server 2012      | Hỗ trợ Multi-channel, mã hóa End-to-End và truy cập lưu trữ từ xa. |
| **SMB 3.0.2**    | Windows 8.1 / Server 2012 R2 | Cải tiến SMB 3.0.                                                  |
| **SMB 3.1.1**    | Windows 10 / Server 2016     | Kiểm tra toàn vẹn dữ liệu (Integrity Checking) và mã hóa AES-128.  |

Lưu ý: SMB 2 và SMB 3 được khuyến nghị sử dụng trong các hệ thống hiện đại vì an toàn và hiệu quả hơn SMB 1.

### Samba và Active Directory

Các phiên bản mới của Samba có thể:

- Tham gia Domain.
- Hoạt động như Domain Member.
- Thậm chí trở thành: Active Directory Domain Controller

Samba sử dụng hai daemon chính:

| Daemon | Chức năng                                                    |
| ------ | ------------------------------------------------------------ |
| `smbd` | Cung cấp dịch vụ SMB (chia sẻ file, thư mục, máy in).        |
| `nmbd` | Cung cấp dịch vụ NetBIOS (đăng ký tên máy, Workgroup, WINS). |

### Workgroup

Các máy SMB thường thuộc một: WORKGROUP

Ví dụ:
```
WORKGROUP

PC01
PC02
FILESERVER
PRINTER01
```

Workgroup giúp các máy nhận biết nhau trong cùng mạng nội bộ.

### NBNS / WINS

Nếu có nhiều máy trong mạng:

Thay vì mỗi máy tự nhớ tên nhau,

sẽ có:

NBNS (NetBIOS Name Server)

sau này phát triển thành:

WINS (Windows Internet Name Service)

Chức năng:

```
Tên máy
      │
      ▼
FILESERVER
      │
      ▼
192.168.1.20
```

Giống như DNS nhưng dành cho NetBIOS.

# Cấu hình mặc định của Samba (SMB)

## 1. File cấu hình Samba

Samba được cấu hình thông qua file:

```bash
/etc/samba/smb.conf
```

File này bao gồm:

* **Global Settings**: Cấu hình chung áp dụng cho toàn bộ SMB Server.
* **Share Settings**: Cấu hình riêng cho từng thư mục chia sẻ (Share).

Ví dụ:

```ini
[global]
    workgroup = DEV.INFREIGHT.HTB
    server string = DEVSMB

[printers]
...

[print$]
...
```

Trong đó, các cấu hình ở từng Share có thể **ghi đè (override)** các cấu hình chung nếu cần.

---

# Các cấu hình quan trọng

| Cấu hình                       | Ý nghĩa                                                                  |
| ------------------------------ | ------------------------------------------------------------------------ |
| `[sharename]`                  | Tên Share được chia sẻ trên mạng.                                        |
| `workgroup`                    | Tên Workgroup hoặc Domain của SMB Server.                                |
| `path`                         | Thư mục thực tế được chia sẻ.                                            |
| `server string`                | Thông tin mô tả hiển thị khi client kết nối.                             |
| `unix password sync = yes`     | Đồng bộ mật khẩu UNIX với mật khẩu SMB.                                  |
| `usershare allow guests = yes` | Cho phép người dùng chưa xác thực (Guest) truy cập Share.                |
| `map to guest = bad user`      | Nếu người dùng đăng nhập không tồn tại thì chuyển thành tài khoản Guest. |
| `browseable = yes`             | Hiển thị Share khi người dùng liệt kê danh sách Share.                   |
| `guest ok = yes`               | Cho phép truy cập Share mà không cần mật khẩu.                           |
| `read only = yes`              | Chỉ cho phép đọc dữ liệu.                                                |
| `create mask = 0700`           | Quyền mặc định của file mới được tạo.                                    |

---

# Các cấu hình nguy hiểm

Một số cấu hình có thể tạo ra rủi ro bảo mật nếu bật trên hệ thống:

| Cấu hình                    | Rủi ro                                                |
| --------------------------- | ----------------------------------------------------- |
| `browseable = yes`          | Cho phép mọi người liệt kê Share trên server.         |
| `read only = no`            | Cho phép sửa và xóa file.                             |
| `writable = yes`            | Cho phép ghi dữ liệu lên Share.                       |
| `guest ok = yes`            | Cho phép truy cập mà không cần xác thực.              |
| `enable privileges = yes`   | Cho phép sử dụng các quyền đặc biệt (SID Privileges). |
| `create mask = 0777`        | File mới có toàn quyền (rất nguy hiểm).               |
| `directory mask = 0777`     | Thư mục mới có toàn quyền.                            |
| `logon script = script.sh`  | Chạy script khi người dùng đăng nhập.                 |
| `magic script = script.sh`  | Tự động thực thi script khi file được đóng.           |
| `magic output = script.out` | Chỉ định nơi lưu kết quả của Magic Script.            |

Nếu các cấu hình như `guest ok = yes`, `writable = yes` và `browseable = yes` được bật cùng lúc, kẻ tấn công có thể:

* Liệt kê toàn bộ Share.
* Truy cập không cần tài khoản.
* Upload hoặc chỉnh sửa file trên Server.

---

# Ví dụ Share

Một Share tên **notes** được cấu hình như sau:

```ini
[notes]
    path = /mnt/notes/

    browseable = yes
    read only = no
    writable = yes
    guest ok = yes

    create mask = 0777
    directory mask = 0777
```

Đây là cấu hình rất nguy hiểm vì:

* Có thể truy cập ẩn danh.
* Có thể xem danh sách file.
* Có thể tải lên hoặc chỉnh sửa file.
* File mới được tạo có quyền `777`.

# Khởi động lại Samba

Sau khi chỉnh sửa cấu hình:

```bash
sudo systemctl restart smbd
```

để áp dụng thay đổi.

---

# Liệt kê các Share

Có thể sử dụng **smbclient** để liệt kê Share:

```bash
smbclient -N -L //10.129.14.128
```

Trong đó:

* `-L`: Liệt kê các Share.
* `-N`: Null Session (không cần username/password).

Ví dụ kết quả:

```text
print$
home
dev
notes
IPC$
```

---

# Kết nối vào Share

Kết nối tới Share:

```bash
smbclient //10.129.14.128/notes
```

Nếu Share cho phép Guest:

```text
Anonymous login successful
```

Sau khi kết nối có thể dùng:

```bash
help
```

để xem toàn bộ lệnh hỗ trợ.

---

# Các lệnh thường dùng

| Lệnh         | Chức năng                                                |
| ------------ | -------------------------------------------------------- |
| `ls`         | Liệt kê file.                                            |
| `cd`         | Chuyển thư mục.                                          |
| `get`        | Tải file từ SMB Server.                                  |
| `put`        | Upload file lên SMB Server.                              |
| `mkdir`      | Tạo thư mục.                                             |
| `rm`         | Xóa file.                                                |
| `help`       | Hiển thị danh sách lệnh.                                 |
| `!<command>` | Chạy lệnh trên máy cục bộ mà không thoát khỏi smbclient. |

Ví dụ:

```bash
get prep-prod.txt
```

Sau đó:

```bash
!cat prep-prod.txt
```

để đọc file ngay trên máy local.

---

# Kiểm tra kết nối SMB

Quản trị viên có thể sử dụng:

```bash
smbstatus
```

để xem:

* Phiên bản Samba.
* Người dùng đang kết nối.
* Địa chỉ IP của Client.
* Share đang truy cập.
* Phiên bản SMB sử dụng.
* Trạng thái Signing và Encryption.
* Các file đang bị khóa.

Đây là công cụ hữu ích để theo dõi hoạt động và phát hiện truy cập bất thường.

# Samba trong Domain

Khi hoạt động trong môi trường **Active Directory**, Samba có thể tham gia Domain như một Domain Member.

Việc xác thực người dùng sẽ do **Domain Controller** thực hiện thông qua cơ sở dữ liệu:

* **NTDS.dit**
* **SAM (Security Account Manager)**

Khi người dùng đăng nhập hoặc truy cập Share, Domain Controller sẽ xác minh tài khoản và quyền truy cập trước khi cho phép kết nối.

# Footprinting the Service

`sudo nmap 10.129.14.128 -sV -sC -p139,445`

`rpcclient -U "" 10.129.14.128`

Các lệnh dưới đây thường được sử dụng với rpcclient để thu thập thông tin từ máy chủ SMB hoặc Active Directory.

| Lệnh                      | Mô tả                                                                                                                              |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `srvinfo`                 | Hiển thị thông tin về máy chủ SMB, như tên máy, hệ điều hành và phiên bản.                                                         |
| `enumdomains`             | Liệt kê tất cả các Domain có trong mạng.                                                                                           |
| `querydominfo`            | Hiển thị thông tin về Domain, máy chủ và các thông tin cơ bản của Domain.                                                          |
| `netshareenumall`         | Liệt kê toàn bộ các Share (thư mục chia sẻ) trên máy chủ SMB.                                                                      |
| `netsharegetinfo <share>` | Hiển thị thông tin chi tiết của một Share cụ thể (đường dẫn, quyền, mô tả...).                                                     |
| `enumdomusers`            | Liệt kê toàn bộ tài khoản người dùng (Domain Users) trong Domain.                                                                  |
| `queryuser <RID>`         | Hiển thị thông tin chi tiết của một người dùng dựa trên **RID (Relative Identifier)**, như tên, mô tả, SID và các thuộc tính khác. |
