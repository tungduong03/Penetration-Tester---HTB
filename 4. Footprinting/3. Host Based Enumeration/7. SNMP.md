# SNMP (Simple Network Management Protocol)

## 1. Tổng quan

**SNMP (Simple Network Management Protocol)** là giao thức dùng để **giám sát (monitoring)** và **quản lý (management)** các thiết bị mạng từ xa.

Các thiết bị thường hỗ trợ SNMP:

* Router
* Switch
* Server
* Firewall
* Printer
* IoT Devices
* Các thiết bị mạng khác

SNMP cho phép:

* Theo dõi trạng thái thiết bị.
* Thu thập thông tin hệ thống.
* Thay đổi cấu hình từ xa (nếu được cấp quyền).

---

## 2. Cổng mặc định

| Port        | Chức năng                                                  |
| ----------- | ---------------------------------------------------------- |
| **UDP 161** | Client gửi yêu cầu và quản lý thiết bị (Query/Set).        |
| **UDP 162** | Thiết bị gửi **SNMP Trap** (thông báo sự kiện) đến Client. |

**SNMP Trap** là cơ chế để thiết bị chủ động gửi cảnh báo (ví dụ: mất kết nối, lỗi phần cứng...) mà không cần Client yêu cầu.

---

## 3. MIB và OID

### MIB (Management Information Base)

MIB là cơ sở dữ liệu mô tả các thông tin mà SNMP có thể truy vấn trên thiết bị.

MIB:

* Được lưu dưới dạng file văn bản (ASN.1).
* Không chứa dữ liệu thực tế.
* Chỉ mô tả vị trí và kiểu dữ liệu của từng thông tin.

---

### OID (Object Identifier)

OID là **định danh duy nhất** của từng đối tượng trong MIB.

Đặc điểm:

* Có dạng cây phân cấp.
* Viết dưới dạng các số ngăn cách bởi dấu chấm.

Ví dụ:

```text
1.3.6.1.2.1.1.5
```

Chuỗi OID càng dài thì thông tin càng cụ thể.

---

# 4. Các phiên bản SNMP

| Phiên bản   | Đặc điểm                                                            |
| ----------- | ------------------------------------------------------------------- |
| **SNMPv1**  | Không xác thực, không mã hóa, dữ liệu truyền dạng Plaintext.        |
| **SNMPv2c** | Có Community String nhưng vẫn truyền Plaintext, không mã hóa.       |
| **SNMPv3**  | Hỗ trợ Username/Password, xác thực và mã hóa dữ liệu, an toàn nhất. |

**Lưu ý:**

* Nhiều hệ thống hiện nay vẫn sử dụng **SNMPv2c** vì việc nâng cấp lên v3 khá phức tạp.
* Đây là nguyên nhân khiến SNMP thường xuất hiện trong các bài pentest.

---

# 5. Community String

Community String giống như một **mật khẩu** để truy cập thông tin SNMP.

Ví dụ:

```text
public
private
monitor
```

Vấn đề:

* Trong SNMPv1/v2c, Community String được gửi **dạng Plaintext**.
* Có thể bị sniff trên mạng.
* Nếu sử dụng giá trị mặc định (`public`, `private`) rất dễ bị khai thác.

---

# 6. Default Configuration

File cấu hình phổ biến trên Linux:

```bash
/etc/snmp/snmpd.conf
```

Một số cấu hình thường gặp:

* `sysLocation`: Vị trí thiết bị.
* `sysContact`: Thông tin quản trị.
* `agentaddress`: Địa chỉ SNMP Agent lắng nghe.
* `view`: Giới hạn OID được phép truy cập.
* `rocommunity`: Community chỉ đọc (Read Only).
* `rwcommunity`: Community đọc/ghi (Read/Write).
* `rouser`: Người dùng của SNMPv3.

---

# 7. Dangerous Settings

Một số cấu hình nguy hiểm:

| Cấu hình                          | Nguy cơ                                                     |
| --------------------------------- | ----------------------------------------------------------- |
| `rwuser noauth`                   | Cho phép **đọc/ghi toàn bộ OID** mà **không cần xác thực**. |
| `rwcommunity <community> <IPv4>`  | Cấp quyền **Read/Write** toàn bộ MIB bằng Community String. |
| `rwcommunity6 <community> <IPv6>` | Tương tự `rwcommunity` nhưng cho IPv6.                      |

Nếu cấu hình sai, kẻ tấn công có thể:

* Đọc toàn bộ thông tin hệ thống.
* Thay đổi cấu hình thiết bị.
* Thu thập thông tin phục vụ tấn công leo thang.
* Kiểm soát một số thiết bị mạng.

# Footprinting the Service

Để thu thập thông tin về SNMP, chúng ta có thể sử dụng các công cụ như `snmpwalk`, `onesixtyone` và `braa`.

## SNMPwalk

`snmpwalk -v2c -c public 10.129.14.128`

## OneSixtyOne

```bash
reDrose18@htb[/htb]$ sudo apt install onesixtyone
reDrose18@htb[/htb]$ onesixtyone -c /opt/useful/seclists/Discovery/SNMP/snmp.txt 10.129.14.128

10.129.14.128 [public] Linux htb 5.11.0-37-generic #41~20.04.2-Ubuntu SMP Fri Sep 24 09:06:38 UTC 2021 x86_64
```

## Braa

```bash
reDrose18@htb[/htb]$ sudo apt install braa
reDrose18@htb[/htb]$ braa <community string>@<IP>:.1.3.6.*   # Syntax
reDrose18@htb[/htb]$ braa public@10.129.14.128:.1.3.6.*
```
