# NFS (Network File System)

## NFS là gì?

**NFS (Network File System)** là giao thức chia sẻ file qua mạng do **Sun Microsystems** phát triển, có mục đích tương tự SMB: cho phép truy cập file/thư mục trên máy từ xa như đang sử dụng trên máy cục bộ.

Tuy nhiên:

* **NFS** chủ yếu dùng cho **Linux/Unix**.
* **SMB** chủ yếu dùng cho **Windows**.
* Hai giao thức này **không tương thích trực tiếp** với nhau.

---

# Các phiên bản NFS

| Phiên bản   | Đặc điểm                                                                                                                                                                   |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **NFSv2**   | Phiên bản cũ, hoạt động chủ yếu qua UDP, vẫn được một số hệ thống hỗ trợ.                                                                                                  |
| **NFSv3**   | Hỗ trợ file lớn hơn, báo lỗi tốt hơn, nhưng không hoàn toàn tương thích với NFSv2. Xác thực dựa trên máy khách (client).                                                   |
| **NFSv4**   | Hỗ trợ Kerberos, ACL, hoạt động tốt qua Firewall/Internet, chỉ sử dụng cổng **2049**, bảo mật và hiệu năng cao hơn. Đây là phiên bản đầu tiên sử dụng cơ chế **stateful**. |
| **NFSv4.1** | Bổ sung **pNFS** (truy cập song song nhiều server), **Multipathing**, phù hợp với hệ thống Cluster và tăng khả năng mở rộng.                                               |

---

# Cách hoạt động

NFS sử dụng giao thức:

* **ONC-RPC (SUN-RPC)**

Hoạt động trên:

* TCP/UDP **111** (RPC)
* TCP/UDP **2049** (NFS)

Trong đó:

* Port **111** dùng để quản lý dịch vụ RPC.
* Port **2049** dùng để truyền dữ liệu NFS.

---

# Cơ chế xác thực

Khác với SMB, NFS **không có cơ chế xác thực riêng**.

Thay vào đó:

* Việc xác thực được thực hiện thông qua **RPC**.
* Phân quyền dựa trên:

  * UID (User ID)
  * GID (Group ID)
  * Group Membership

Ví dụ:

```text
User A (UID 1000)
        │
        ▼
NFS Server
        │
        ▼
Kiểm tra UID/GID để quyết định quyền truy cập
```

### Nhược điểm

Nếu:

* Client có UID = 1000 là **alice**
* Server có UID = 1000 là **bob**

thì Server vẫn xem đó là cùng một UID.

Do đó, NFS chỉ nên sử dụng trong **mạng nội bộ đáng tin cậy (Trusted Network)**.

# File cấu hình

NFS được cấu hình trong file:

```bash
/etc/exports
```

File này định nghĩa:

* Thư mục nào được chia sẻ.
* Máy hoặc mạng nào được phép truy cập.
* Quyền truy cập.

Ví dụ:

```bash
/mnt/nfs 10.129.14.0/24(sync,no_subtree_check)
```

Ý nghĩa:

* Chia sẻ thư mục **/mnt/nfs**
* Cho toàn bộ subnet **10.129.14.0/24**
* Sử dụng các tùy chọn:

  * `sync`
  * `no_subtree_check`

Sau khi chỉnh sửa:

```bash
systemctl restart nfs-kernel-server
```

để áp dụng cấu hình.

---

# Các tùy chọn thường dùng

| Tùy chọn           | Ý nghĩa                                                                                                   |
| ------------------ | --------------------------------------------------------------------------------------------------------- |
| `rw`               | Cho phép đọc và ghi.                                                                                      |
| `ro`               | Chỉ cho phép đọc.                                                                                         |
| `sync`             | Ghi dữ liệu đồng bộ (an toàn hơn nhưng chậm hơn).                                                         |
| `async`            | Ghi dữ liệu bất đồng bộ (nhanh hơn nhưng kém an toàn hơn).                                                |
| `secure`           | Chỉ cho phép kết nối từ các cổng < 1024.                                                                  |
| `insecure`         | Cho phép kết nối từ các cổng > 1024.                                                                      |
| `no_subtree_check` | Không kiểm tra cây thư mục con, giúp tăng hiệu năng.                                                      |
| `root_squash`      | Chuyển quyền của user **root** thành user **anonymous**, ngăn root trên client có toàn quyền trên server. |

---

# Các cấu hình nguy hiểm

Một số cấu hình có thể gây rủi ro bảo mật:

| Tùy chọn         | Rủi ro                                                                                                                                                    |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `rw`             | Cho phép ghi dữ liệu lên NFS Share.                                                                                                                       |
| `insecure`       | Cho phép sử dụng các cổng >1024 để kết nối, dễ bị lạm dụng hơn.                                                                                           |
| `nohide`         | Cho phép hiển thị cả các filesystem được mount bên dưới thư mục chia sẻ.                                                                                  |
| `no_root_squash` | Không chuyển quyền của root thành anonymous, khiến **root trên client vẫn là root trên server**, có thể dẫn đến chiếm quyền hoặc sửa đổi toàn bộ dữ liệu. |

### Đặc biệt nguy hiểm: `no_root_squash`

Đây là cấu hình nguy hiểm nhất.

Thông thường:

```text
Client Root
      │
      ▼
Anonymous User trên Server
```

Nếu bật:

```text
no_root_squash
```

thì:

```text
Client Root
      │
      ▼
Server Root
```

Điều này cho phép root trên máy client có toàn quyền trên NFS Share, tạo ra nguy cơ leo thang đặc quyền và chiếm quyền hệ thống.

# Ý chính cần nhớ

* **NFS** là giao thức chia sẻ file dành cho **Linux/Unix**, tương tự SMB nhưng sử dụng giao thức và cơ chế hoạt động khác.
* **NFSv4** cải thiện đáng kể về bảo mật và hiệu năng, hỗ trợ Kerberos, ACL và chỉ sử dụng **port 2049**.
* NFS xác thực dựa trên **UID/GID**, không có cơ chế xác thực người dùng riêng nên chỉ phù hợp với **mạng tin cậy**.
* Toàn bộ cấu hình được quản lý trong **`/etc/exports`**, nơi định nghĩa thư mục chia sẻ, phạm vi truy cập và quyền.
* Trong Pentest, cần đặc biệt chú ý các cấu hình **`rw`**, **`insecure`** và đặc biệt là **`no_root_squash`**, vì chúng có thể dẫn đến việc ghi dữ liệu trái phép hoặc leo thang đặc quyền lên quyền **root**.

# Footprinting the Service

Khi thu thập thông tin về NFS, các cổng TCP 111 và 2049 là rất cần thiết. Chúng ta cũng có thể lấy thông tin về dịch vụ NFS và máy chủ thông qua RPC, như ví dụ bên dưới.

`sudo nmap 10.129.14.128 -p111,2049 -sV -sC`

Dùng NSE script của nmap 

`sudo nmap --script nfs* 10.129.14.128 -sV -p111,2049`

Hiển thị các cổ phiếu NFS hiện có

`showmount -e 10.129.14.128`

Mounting NFS Share

```bash
reDrose18@htb[/htb]$ mkdir target-NFS
reDrose18@htb[/htb]$ sudo mount -t nfs 10.129.14.128:/ ./target-NFS/ -o nolock
reDrose18@htb[/htb]$ cd target-NFS
reDrose18@htb[/htb]$ tree .

.
└── mnt
    └── nfs
        ├── id_rsa
        ├── id_rsa.pub
        └── nfs.share

2 directories, 3 files
```

List Contents with Usernames & Group Names

`ls -l mnt/nfs/`

List Contents with UIDs & GUIDs

`ls -n mnt/nfs/`

Điều quan trọng cần lưu ý là nếu tùy chọn `root_squash` được thiết lập, chúng ta không thể chỉnh sửa tệp `backup.sh` ngay cả khi có quyền root.

Sau khi đã thực hiện tất cả các bước cần thiết và thu thập được thông tin cần thiết, chúng ta có thể gỡ bỏ phân vùng chia sẻ NFS.

Unmounting

```bash
reDrose18@htb[/htb]$ cd ..
reDrose18@htb[/htb]$ sudo umount ./target-NFS
```
