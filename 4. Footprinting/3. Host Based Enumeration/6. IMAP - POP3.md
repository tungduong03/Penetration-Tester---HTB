# IMAP / POP3

## 1. Tổng quan

**IMAP (Internet Message Access Protocol)** và **POP3 (Post Office Protocol v3)** là hai giao thức dùng để **nhận email** từ Mail Server.

* **SMTP** dùng để **gửi email**.
* **IMAP/POP3** dùng để **nhận email**.

---

## 2. IMAP và POP3 khác nhau như thế nào?

### IMAP

IMAP cho phép người dùng quản lý email **trực tiếp trên Mail Server**.

Đặc điểm:

* Email vẫn được lưu trên Server cho đến khi bị xóa.
* Hỗ trợ thư mục (Inbox, Sent, Drafts, Archive...).
* Đồng bộ giữa nhiều thiết bị (PC, điện thoại, tablet...).
* Nhiều người dùng có thể truy cập cùng một mailbox.
* Có thể xem, tìm kiếm và quản lý email ngay trên Server.
* Một số ứng dụng hỗ trợ chế độ Offline và sẽ đồng bộ khi có mạng.

**Port mặc định:**

* 143 (IMAP)
* 993 (IMAPS - SSL/TLS)

---

### POP3

POP3 đơn giản hơn IMAP.

Chỉ hỗ trợ:

* Liệt kê email.
* Tải email về máy.
* Xóa email trên Server.

Đặc điểm:

* Không hỗ trợ thư mục.
* Không đồng bộ giữa nhiều thiết bị.
* Phù hợp khi chỉ sử dụng một thiết bị để đọc email.

**Port mặc định:**

* 110 (POP3)
* 995 (POP3S - SSL/TLS)

---

## 3. Bảo mật

Nếu không sử dụng SSL/TLS:

* Username truyền dạng Plaintext.
* Password truyền dạng Plaintext.
* Nội dung email cũng có thể bị nghe lén.

Vì vậy hầu hết Mail Server hiện nay sử dụng:

* IMAPS (993)
* POP3S (995)

để mã hóa kết nối.

# Default Configuration

Hai dịch vụ phổ biến trên Linux là:

* `dovecot-imapd`
* `dovecot-pop3d`

Dovecot có rất nhiều tùy chọn cấu hình và thường được dùng để triển khai IMAP/POP3 Server.

---

# Các lệnh IMAP thường dùng

| Lệnh                      | Chức năng                                          |
| ------------------------- | -------------------------------------------------- |
| `LOGIN username password` | Đăng nhập.                                         |
| `LIST "" *`               | Liệt kê tất cả mailbox/thư mục.                    |
| `CREATE "INBOX"`          | Tạo mailbox mới.                                   |
| `DELETE "INBOX"`          | Xóa mailbox.                                       |
| `RENAME`                  | Đổi tên mailbox.                                   |
| `LSUB`                    | Liệt kê mailbox đã subscribe.                      |
| `SELECT INBOX`            | Chọn mailbox để thao tác.                          |
| `UNSELECT`                | Thoát mailbox hiện tại.                            |
| `FETCH <ID> all`          | Đọc email theo ID.                                 |
| `CLOSE`                   | Đóng mailbox và xóa các email đã đánh dấu Deleted. |
| `LOGOUT`                  | Đăng xuất.                                         |

---

# Các lệnh POP3 thường dùng

| Lệnh        | Chức năng                             |
| ----------- | ------------------------------------- |
| `USER`      | Nhập username.                        |
| `PASS`      | Nhập password.                        |
| `STAT`      | Hiển thị số lượng email.              |
| `LIST`      | Liệt kê email và kích thước.          |
| `RETR <ID>` | Tải email theo ID.                    |
| `DELE <ID>` | Xóa email.                            |
| `CAPA`      | Hiển thị các tính năng server hỗ trợ. |
| `RSET`      | Hủy các thay đổi trước đó.            |
| `QUIT`      | Đóng kết nối.                         |

# Dangerous Settings

Một số cấu hình sai có thể làm rò rỉ nhiều thông tin quan trọng, đặc biệt trên các Mail Server tự quản lý (self-hosted).

| Cấu hình                  | Nguy cơ                                                           |
| ------------------------- | ----------------------------------------------------------------- |
| `auth_debug`              | Ghi log toàn bộ quá trình xác thực.                               |
| `auth_debug_passwords`    | Ghi lại cả mật khẩu trong log.                                    |
| `auth_verbose`            | Ghi chi tiết các lần đăng nhập thất bại và nguyên nhân.           |
| `auth_verbose_passwords`  | Ghi lại mật khẩu khi xác thực (có thể bị cắt bớt).                |
| `auth_anonymous_username` | Cho phép hoặc hỗ trợ đăng nhập Anonymous nếu cấu hình không đúng. |

Nếu các log này bị lộ hoặc người dùng không được phân quyền đúng, kẻ tấn công có thể thu thập:

* Username.
* Password.
* Thông tin xác thực.
* Nội dung email.
* Dữ liệu nhạy cảm trong hộp thư.

# Footprinting the Service

Theo mặc định, cổng 110 và 995 được sử dụng cho POP3, và cổng 143 và 993 được sử dụng cho IMAP. Các cổng cao hơn (993 và 995) sử dụng TLS/SSL để mã hóa quá trình liên lạc giữa máy khách và máy chủ. Sử dụng Nmap, chúng ta có thể quét máy chủ để tìm các cổng này.

```bash
sudo nmap 10.129.14.128 -sV -p110,143,993,995 -sC
```

Nếu có user:password 

```bash
curl -k 'imaps://10.129.14.128' --user cry0l1t3:1234 -v
```

Để tương tác với máy chủ IMAP hoặc POP3 qua SSL, chúng ta có thể sử dụng openssl, cũng như ncat. Các lệnh cho việc này sẽ trông như sau:

```bash
openssl s_client -connect 10.129.14.128:pop3s
```

```bash
openssl s_client -connect 10.129.14.128:imaps
```
