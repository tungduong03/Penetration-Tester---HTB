# MSSQL (Microsoft SQL Server)

## Tổng quan

**Microsoft SQL Server (MSSQL)** là hệ quản trị cơ sở dữ liệu quan hệ (RDBMS) của Microsoft. Khác với **MySQL** (mã nguồn mở), MSSQL là **phần mềm mã nguồn đóng (closed source)** và ban đầu chỉ chạy trên **Windows** (hiện nay đã hỗ trợ Linux và macOS).

MSSQL được sử dụng rộng rãi trong các ứng dụng phát triển trên nền tảng **.NET Framework** nhờ khả năng tích hợp tốt với hệ sinh thái Microsoft.

- **Cổng mặc định:** TCP **1433**
- **Ngôn ngữ truy vấn:** T-SQL (Transact-SQL)

---

# MSSQL Clients

Có nhiều công cụ dùng để quản trị hoặc kết nối đến MSSQL.

### SQL Server Management Studio (SSMS)

Đây là công cụ chính thức của Microsoft dùng để:

- Quản trị database
- Backup/Restore
- Quản lý user
- Thực hiện truy vấn SQL

Lưu ý:

- SSMS không nhất thiết cài trên máy chủ Database.
- Có thể được cài trên máy quản trị.
- Nếu pentest chiếm được máy có SSMS thì có thể lấy được **credential đã lưu** để truy cập Database.

### Một số client phổ biến

- SQL Server Management Studio (SSMS)
- mssql-cli
- SQL Server PowerShell
- HeidiSQL
- SQLPro
- **Impacket mssqlclient.py** (phổ biến nhất trong pentest)

Kiểm tra vị trí:

```bash
locate mssqlclient
```

Ví dụ:

```text
/usr/bin/impacket-mssqlclient
```

# Database hệ thống mặc định

| Database | Chức năng |
|-----------|-----------|
| **master** | Lưu toàn bộ thông tin hệ thống của SQL Server |
| **model** | Database mẫu, mọi database mới đều được tạo dựa trên đây |
| **msdb** | Chứa thông tin SQL Server Agent (Job, Backup, Alert...) |
| **tempdb** | Chứa dữ liệu và đối tượng tạm thời |
| **resource** | Chứa các đối tượng hệ thống (Read-only) |

# Cấu hình mặc định

Sau khi cài đặt:

- MSSQL thường chạy dưới tài khoản:

```
NT SERVICE\MSSQLSERVER
```

- Cho phép **Windows Authentication**
- **Không bắt buộc mã hóa (Encryption)** khi client kết nối (mặc định)

Windows Authentication nghĩa là:

- Hệ điều hành Windows xử lý việc xác thực.
- Có thể xác thực qua:
  - Local SAM
  - Active Directory

Ưu điểm:

- Quản lý tập trung.
- Audit dễ dàng.

Nhược điểm:

Nếu tài khoản Windows bị chiếm thì attacker có thể:

- Truy cập MSSQL
- Leo thang đặc quyền
- Di chuyển ngang (Lateral Movement)

---

# Cấu hình nguy hiểm

Một số cấu hình thường gặp trong thực tế có thể gây rủi ro:

## 1. Không mã hóa kết nối

Client kết nối MSSQL mà không sử dụng TLS.

→ Có thể bị nghe lén (MITM).

---

## 2. Self-signed Certificate

Nếu dùng TLS nhưng sử dụng **Self-signed Certificate**.

→ Có thể bị giả mạo chứng chỉ.

---

## 3. Named Pipes

Cho phép kết nối thông qua:

```
\\HOST\pipe\sql\query
```

Trong một số trường hợp có thể hỗ trợ lateral movement hoặc relay attack.

---

## 4. Tài khoản `sa`

Nguy hiểm nếu:

- Không đổi mật khẩu
- Dùng mật khẩu yếu
- Không disable sau khi cài đặt

# Footprinting MSSQL

## Quét bằng Nmap

Port mặc định:

```
1433/TCP
```

Ví dụ:

```bash
sudo nmap -sV -p1433 --script ms-sql-info 10.10.10.10
```

Hoặc sử dụng nhiều script:

```bash
sudo nmap \
--script ms-sql-info,\
ms-sql-empty-password,\
ms-sql-config,\
ms-sql-ntlm-info,\
ms-sql-tables,\
ms-sql-dump-hashes \
-p1433 10.10.10.10
```

---

## Thông tin có thể thu được

Các NSE Script của Nmap có thể giúp lấy:

- Hostname
- Instance Name
- MSSQL Version
- Service Pack
- Named Pipe
- TCP Port
- Kiểm tra mật khẩu rỗng
- Kiểm tra quyền truy cập database
- Cấu hình MSSQL
- NTLM Information
- Hash (nếu có quyền)

Ví dụ:

```
Instance Name:
MSSQLSERVER

Version:
Microsoft SQL Server 2019

Named Pipe:
\\10.10.10.10\pipe\sql\query

TCP:
1433
```

Đây đều là thông tin rất hữu ích khi enumerate.

# Kết nối MSSQL với Mssqlclient.py

Nếu có credential:

Có thể dùng **Impacket mssqlclient.py**

```bash
python3 mssqlclient.py Administrator@10.10.10.10 -windows-auth
```

Sau khi nhập password thành công sẽ kết nối trực tiếp tới SQL Server.

Nếu server yêu cầu mã hóa:

```
Encryption required, switching to TLS
```

Impacket sẽ tự chuyển sang TLS.
